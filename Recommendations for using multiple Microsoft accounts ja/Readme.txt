[記事] 複数 Microsoft アカウント運用の勧め

## BitLocker 回復キーの問題

BitLocker 回復キーは、次の性質を持っています：

- PC 本体とは独立した「BitLocker 復旧の最後の砦」すなわち1個しか無い鍵
- 紛失すると復号できず、データは完全に失われる
- Microsoft アカウントに自動バックアップされるが、1つのアカウントに依存されるため、様々なリスクがある。
- ファイルやなど紙でも保管できるが、セキュリティと運用の面から現実的では無い。

## 複数 Microsoft アカウント作成手順

- PC に Microsoft アカウント A でサインイン
- BitLocker を有効化（自動で A に回復キーが保存される）
- ローカルアカウントに切り替え
- Microsoft アカウント B でサインイン
- デバイス登録が行われる
- 手作業で「回復キーを Microsoft アカウントに保存」を実行
- B のアカウントにも回復キーが保存される

※重要：この手順は Windows のアカウント作成、サインインアカウント切り替えと、デバイス登録の仕組みを理解している個人向けです。
ここまでの説明が理解出来ない方にはお勧めしません。

## なぜ複数の Microsoft アカウントに保管するのが有効なのか

1. アカウントロックのリスクを分散できる
  - パスワード忘れ
  - 多要素認証のトラブル

※補足： ここの質問でも時々ある、アカウント停止（誤検知含む）1 つの Microsoft アカウントに依存していると、アカウントに入れない＝回復キーにアクセスできないという最悪の状況が起こり得ます。

2. 管理者・自分の別アカウント・実験用など、複数の視点で管理できる

例えば1台のPCに、「個人用」「常用」「評価用」「緊急用」など複数アカウントを持つ人は、
いくつかの Microsoft アカウントに回復キーを保管しておくことで、どれか 1つが使えなくても復旧できるという冗長性による保全が可能です。

3. Microsoft アカウントのデバイス管理画面は、アカウントごとに独立
つまり、
- アカウント A にログイン → PC のデバイス、回復キーが見える
- アカウント B にログイン → 同じ PCデバイス の回復キーが見える
という状態を作れるのがポイントです。

運用は少しだけ手間ですが、ファイルや紙などでの回復キー保管よりもはるかに合理的、強力な運用方法だと思いませんか？
